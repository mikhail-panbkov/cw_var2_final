# Курсовая работа по системному программированию, вариант 2

Вариант 2: индекс для полей `INDEXED` реализован как B+-tree.

## Сборка

```powershell
cmake -G Ninja -S . -B build
cmake --build build
```

Собираются три исполняемых файла: `cwdb` (автономный режим), `cwdb-server` (сервер БД) и `cwdb-client` (терминал-клиент).

## Запуск

### Автономный режим (in-process)

Интерактивно:

```powershell
.\build\cwdb.exe
```

Пакетно:

```powershell
.\build\cwdb.exe examples\variant2_demo.sql
```

### Клиент-серверный режим

В одном окне запустить сервер:

```powershell
.\build\cwdb-server.exe 127.0.0.1 5429
```

В другом окне — клиент (интерактивно или пакетно):

```powershell
.\build\cwdb-client.exe --host 127.0.0.1 --port 5429
.\build\cwdb-client.exe --host 127.0.0.1 --port 5429 examples\variant2_demo.sql
```

Сервер и клиент обмениваются длинно-префиксированными кадрами по TCP (Winsock на Windows, BSD sockets на POSIX). Сервер обслуживает клиентов последовательно; состояние БД, пул интернированных строк и счётчик телеметрии общие для всех подключений. `EXIT;` (или `QUIT;`) закрывает только текущее соединение, сервер продолжает работу.

Данные хранятся в каталоге `data`. Для каждого запроса ведется журнал `data/access.log` (поля `request_id`, `transaction_id`, `start`, `end`, `client`, `handler`, `code`, `status`, `query`). Состояния баз для `REVERT` сохраняются в `_db_history.log` внутри каталога базы; REVERT восстанавливает конкретную таблицу из ближайшего по времени снимка.

Для каждого столбца `INDEXED` B+-дерево сохраняется на диск в файл `index_<столбец>.bpt` внутри каталога таблицы. Файл перезаписывается после каждой мутации (`CREATE TABLE`, `INSERT`, `UPDATE`, `DELETE`) и пересоздаётся после `REVERT`, поэтому всегда отражает актуальные данные. Формат — текстовый дамп структуры дерева (узлы в порядке обхода в ширину: разделяющие ключи и потомки для внутренних узлов, пары `ключ=строка` и связь `next` для листьев).

## Примечания по SQL

- Поддерживаемые типы столбцов: `int` и `string`.
- Операторы сравнения: `==`, `!=`, `<`, `>`, `<=`, `>=`.
- `LIKE` использует регулярные выражения (`std::regex`), поэтому для любой строки нужно писать `LIKE ".*"`, а не `LIKE "*"`.
- `BETWEEN val1 AND val2` задает полуинтервал `[val1, val2)`.
- В сценариях поддерживаются комментарии `//`, `--` и `/* ... */`.
- Завершение интерактивного сеанса: `EXIT;` или `QUIT;`.
- Подсистема телеметрии доступна командой `SHOW TELEMETRY;` (текущий RPS, средний/максимальный RPS за 10 минут, среднее время обработки за 10 секунд, error rate за 1 минуту).
- Пакетный режим выполняет файл и завершает процесс, как требуется в задании.
